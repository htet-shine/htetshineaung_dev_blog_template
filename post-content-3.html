Introduction

Welcome to my website.
This is one of the first php projects I've created during my time studying web development. I wanted to create an online store with cash on delivery system. This project includes, browsing categories, products, shopping cart and checkout functionalities for the frontend and basic admin dashboard for the backend.

I've spent hours upon hours searching for things that I wanted to do that I didn't learn in classes (not in great details anyway). Those includes ajax calls, responsive design, single file for php scripts, etc.

I relied on jquery in this project for my ajax calls, some minor dom manipulations, datatables and popovers. Then I've learned that jquery is going out for better js frameworks.

Anyway, it was a good learning experience. Lets see the scope in next section.

Scope

  • Users will be able to
    • browse products, categories
    • use shopping cart
    • order products with cash on delivery payment method

  • Admins will be able to
    • see overview of data
    • add, edit, delete products
    • add, edit, delete categories
    • add, edit, delete sub-categories
    • add, edit, delete brands
    • add, edit, delete delivery location and fees
    • manage orders
    • update stock

Database Structure

Image: Database

Database has seven tables.
As you can see in the image, a category, subcategory and brand can have many products but a product can have only one category.
A category can have many subcategories but a subcategory can have only one category.
An order can have many products, a product can have many orders. So, a join table "order_product" is created that store the IDs of orders and products.
An order can only have one delivery location but a delivery location can have many orders.

User Guide

In this section, I will walk you through how to use the application. First, lets start with installation.


Installation

If you already know how to set it up, download the project files and skip this part.

  • You need to download the project files here.
  • And xampp here.

  • Install xampp and run Apache and MySQL.
  • Go to "localhost/phpmyadmin" url in your browser and create a new database.
  • Extract the download project zip file and indside "\SQL Setup\", you will find "DB_Backup.sql". Import that file into your newly created database.
  • After that, open "\admin\config\config.php" with notepad or any text editor and change the following 
    $dbuser = "[your user name]";
$dbpass = "[your user password, default is blank]";
$dbname = "[your database name you created earlier]";
  • Copy the entire project folder to "xampp\htdocs" folder
  • Open the browser and type in "localhost/[the project folder name]" and you will see the home page of the website.

Conclusion

First of all, is this project practical? No, its not. One of the biggest reason for that is security. I did not know how to use PHP PDO at the time of developing this project so SQL Injections attacks are welcome. And so many other security flaws.
Secondly, this kind of coding php is incredibly difficult and time consuming to read and debug for anyone.

But what this project did give me is the learning experience and knowledge about

  • php
  • sql queries
  • jquery (dom manipulation, ajax calls, form validations, etc)
  • responsive design
  • many other ui/ux elements
