Introduction

Hi. Welcome to my website.
This is my very first web application that I created using Laravel PHP framework. This is a very simple blog which lack a lot of practical features. But it helped beginners like me (who come from writing only procedural PHP) understand how developing with a framework works. So this little app will cover some of the very basics of dynamic websites, simple CRUD operations. I will explain the scope of the app in next section.

Scope

  • Users will be able
    • create accounts and login
    • create new and edit existing categories
    • create, edit and delete thier own posts
    • comment on posts

Database Structure

Database

Database has four tables and a few other default ones created by laravel. A category and user can have no or many posts. A post can have many or no comments.

Implementation

This section will include brief explanation of my development process. I'm not going to explain line by line as it will make this blog post way too long for anyone's attention span.

Laravel Installation

I do not like installing laravel globally because I like to contain everything in a project folder and choose the versions I want. So, "cd" into a folder and run the following command. 

composer create-project --prefer-dist laravel/laravel [folder name]

Then I've "cd" into the root folder and run following commands. 

php artisan make:model Post -m
php artisan make:seeder PostsTableSeeder
php artisan make:controller PostController

First commnad will create a Post Model and database migration which will be use to create posts table in database.
Second one will create a seeder file that I will use to insert data into database.
Third one will create a Controller for Post where I will code all the functions that will interact with views which in turn interacts with users.


After that I will create all of the above post categories and comments by running the following commands. 

php artisan make:model Category -m
php artisan make:seeder CategoriesTableSeeder
php artisan make:controller CategoryController

php artisan make:model Comment -m
php artisan make:seeder CommentsTableSeeder
php artisan make:controller CommentController

I will be utilizing laravel's default layout and authentication function. So I will run this following command.


For laravel verions below 6.0

php artisan make:auth

For laravel version 6.0. See laravel documentation Here. 

composer require laravel/ui --dev
php artisan ui vue --auth

This will create User Model, Home Controller, users migration, default layout and view files and routes.


These are the relations between post and category, user and comment.


A category can have many posts
A user can have many posts
A post can have many comments


By having relationship functions, I can get any category, user and comment related to posts.


Then I will create views for posts inside a seperate folder named posts.

  • index.blade.php
  • view.blade.php
  • add.blade.php
  • edit.blade.php

Then go to routes and create routes for each one. Then go to PostController and create all the functions for each view and test if it return the correct view.


Then I will add all the attributes already planned in database structure to according migrations. And add pre-made data in according seeders.


After that create a database in my localhost and run following commands in my root directory. 

php artisan migrate
php artisan db:seed

First command create the tables and second one insert the data.


Here is functions in Post Controller. 

middleware: auth
index()
view($id)
add()
create()
edit($id)
update($id)
delete($id)
  • auth middleware allow only index and view functions to be acessed by unauthenticated users. This is to prevent posts having no authors. That can make pretty much every page break.
  • index function will return all the posts ordered by date and limit 6 posts per page
  • view function will take post id as a parameter and return a post matching that id
  • add function will check if user is logged in. If so return add new post form along with all categories to choose from. Else redirect to login page.
  • create function will check data in all input fields are present. If so insert post data into the table. Else return error message.
  • edit function will take post id and check if authenticated user id is matched with user id of the post matching received post id. If so return edit post form. Else return a message.
  • update function will check if data in all input fields are present. If so it will take post id and update the data of the post matching received post id. Else return a message.
  • delete function will take post id check if authenticated user id is matched with user id of the post matching received post id. If so delete the post from the table. Else return a message.

Here is functions in Category Controller.

index()
add()
create()
edit($id)
update($id)
  • index function will return all the categories
  • add function will return add new category form
  • create function will check data in all input fields are present. If so insert post data into the table. Else return error message.
  • edit function will take category id and return edit form with data matching received category id.
  • update function will check if data in all input fields are present. If so it will take category id and update the data of the category matching received post id. Else return error message.

Here is functions in Comment controller

create()
  • create function will insert comment into the table with the post id received from input fields.

User Guide

This section will guide you step by step on how to use the application. If you want to know how to install it click here check out my github repository. There is a detail step by step guide.

Home Screen

Once you setup everything on local server as shown in my github repo, you should be seeing the something similar to the image 1 in your browser.


Image 1: Home Screen

You can see top navigation bar and 6 posts per page. You can use the number buttons at the bottom of the page to navigate between pages. You can click on the post title to see full post as shown in the image below.


Image 2: Single Post

Adding New Post

If you are not logged in, you will be asked to do so in order to submit posts. To login, you need a user account. Click on "Register" in the navigation bar. You will see the image below.


Image 3: Register User

Type in the information and click register. (Note that password must include 8 characters). After that you can go back to login page by clicking "Login" in the navigation bar, type in your email and password and click login. Now you can use all features of the site.

Lets start with adding post by clicking "Add New Post". Your will see a page with big title "Add New Post"


Image 4: Add New Post

You can type in your post content, choose category and click "Add New". You will be redirected back to home page with the success message. (Data must be present in all input fields, otherwise you will see an error message when clicking the button)


Image 5: Post Added

Commenting on Posts

You can comment on posts by going to single post page. As shown in the image below, you will see both your comment and a message if your comment is submitted successfully.


Image 6: Comment

Editing and Deleting Posts

Now you can edit or delete posts that are submitted by you. To edit click on your post title and you will be directed to single post page. There you will find edit and delete button to the right of the post title. Once you click on edit, you will see edit post page. You can change what you want to change and click "confirm". Your post will be updated.


Image 7: Edit

Similar Process for deleting. Click delete instead of edit and your post will be deleted.

You can also add and edit categories in the same way you add posts. See the Images below.


Image 8: Adding Categories

Conclusion

Since this is a project purly focused on getting used to laravel, a lot of good UI/UX guidelines are left out. Such as no confirmation dialog when deleting post. If this is a real blog, it can be very costly such as users accidentally clicking "delete" button right beside "edit" and losing their entire post. And no post content check such as "post title must be below such amount of characters".

I've learned how easier it is to find and fix errors during development compare to normal procedural php. And more importantly laravel covers a lot of basic security features by default such as SQL injections and Cross-Site Request Forgery (CSRF). Overall this project was a good learning experience. Thank You for reading through all of this and please check out my other projects if you have time.
